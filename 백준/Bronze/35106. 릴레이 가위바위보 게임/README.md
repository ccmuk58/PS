### 문제 설명

<p>올해 넥슨은 대학생들을 위한 새로운 대회 NYPC 코드배틀을 개최했다. 코드배틀은 주어진 게임 환경에서 승리할 수 있는 AI 코드를 개발해 서로 대결하는 전략적 팀 프로그래밍 대회이다.</p>

<p>철수는 코드배틀 대회 연습을 위해 릴레이 가위바위보 게임을 진행했다. 이 게임은 총 $3N$번의 라운드를 진행하며, 각 라운드에서 참가자는 손동작에 해당하는 번호를 정수로 출력해야 한다. 게임에서 사용되는 손동작은 다음과 같다.</p>

<ul>
<li>$1$: 가위</li>
<li>$2$: 바위</li>
<li>$3$: 보</li>
</ul>

<p>게임이 모두 끝난 뒤, 참가자가 세 손동작을 정확히 $N$번씩 출력했다면 추가 점수를 획득한다.</p>

<p>철수는 추가 점수를 획득하기 위해 가위, 바위, 보를 각각 $N$번씩 출력하도록 프로그램을 작성했다. 하지만 프로그램을 실행한 결과, <strong>정확히 한 라운드에서</strong> 오류가 발생하여 원래 출력되어야 할 손동작과 다른 손동작이 출력되었고, 그로 인해 추가 점수를 얻지 못했다는 사실을 알게 되었다.</p>

<p>철수의 프로그램이 출력한 $3N$개의 손동작이 주어졌을 때, 오류가 발생한 라운드에서 원래 출력되었어야 할 손동작과 실제로 출력된 손동작의 번호를 구해보자.</p>

### 입력 

 <p>첫 번째 줄에 정수 $N$이 주어진다. $(1 \leq N \leq 1\,000)$</p>

<p>두 번째 줄에 철수의 프로그램이 출력한 손동작을 의미하는 $3N$개의 정수가 공백으로 구분되어 주어진다. $i$번째 정수는 $i$번째 라운드에 출력한 손동작의 번호로, $1$, $2$, $3$ 중 하나이다.</p>

<p>입력은 정확히 한 라운드에서만, 원래 출력되어야 할 손동작과 다른 손동작이 출력된 경우로만 주어진다.</p>

### 출력 

 <p>첫 번째 줄에 오류가 발생한 라운드에서 원래 출력되었어야 할 손동작의 번호를 출력하고, 두 번째 줄에 그 라운드에서 실제로 출력된 손동작의 번호를 출력한다.</p>
